# 可行性结论与优化方案 — Windows 极简家具自动抠图工具

## 一、可行性结论：✅ 可行

| 关键风险 | 评估 | 结论 |
|---|---|---|
| BiRefNet_HR-matting 纯 CPU / float32 推理 | 模型约 800MB FP32；i5-12400 + 32GB 内存充裕；1024² 输入单张约 30s–2min | 可行（用户明确"质量优先于速度"） |
| 4K 手机照片处理 | 模型只在 1024² 推理，Alpha 再映射回 4K；峰值内存可控（4K RGBA≈50MB） | 可行 |
| GUI 不卡死 | 模型加载/推理全部在 QThread 后台；主线程仅刷新 | 可行 |
| 离线运行 | 首次 HF 下载缓存后离线可用；支持手动选本地模型目录 | 可行 |
| 依赖兼容 | Python 3.11 + CPU PyTorch + PySide6 + transformers(trust_remote_code) 是成熟组合 | 可行，需锁版本 |
| 无 CUDA 环境 | 全程 `device=cpu`，禁止 fp16/cuda | 可行 |

**唯一需要在实现时现场核实的高风险点**：BiRefNet_HR-matting 的**官方预处理与输出解析**。规范明确要求"核对官方、不要凭空编造"。实现 `birefnet_engine.py` 前必须先读 HuggingFace 模型卡 + BiRefNet GitHub 仓库的 `inference.py`，确认：输入尺寸、mean/std、是否 letterbox、输出是否需 sigmoid、Alpha 通道范围与形状。本文档据此给出默认实现策略，但以官方为准。

## 二、对原方案的修正与优化（重点）

### 修正 1：预处理采用 letterbox（非官方的 squash-resize）
**冲突点**：官方 BiRefNet 推理常用 `Resize((1024,1024))` 直接拉伸；但规范明令"禁止直接拉伸导致家具变形"，并列出 letterbox/padding 为推荐方式，且要求"记录 padding 并去除"。

**决议**：采用 **letterbox**——按长边缩放到模型输入尺寸（默认 1024），短边 pad 到正方形（constant pad，值取 ImageNet 均值以减少边缘伪影），记录 padding；推理后先裁掉 padding，再把 Alpha 高质量缩放回原图尺寸。
- 理由：规范明确授权 letterbox；对极端长宽比更稳；满足"输出无比例偏移"。
- 与顾问复核一致。

### 修正 2：OpenCV 用 headless 版本
`opencv-python` 在 Windows 上与 PySide6 共存易引发 DLL 冲突。改用 `opencv-python-headless`。功能仅用 resize/warp，无需 GUI 模块。

### 修正 3：补 HuggingFace 镜像支持（国内首次下载）
模型 ~800MB，国内直连易超时。在 `install.bat` 与设置中支持 `HF_ENDPOINT=https://hf-mirror.com`（默认开启，可在设置关闭）。下载用 `snapshot_download` 带**真实进度回调**（MB/百分比/速度），而非笼统"下载中"。

### 修正 4：模型加载与推理分离的状态机
规范只要求"模型加载/推理后台线程"。优化为显式状态：`未加载 → 加载中(带进度) → 就绪 → 推理中 → 就绪/失败`。模型在 worker 线程内**只加载一次并常驻**；禁止每图重建。

### 修正 5：拖拽打开图片
验收标准写"拖入或打开家具照片"。补窗口拖放接收图片文件（`dragEnterEvent`/`dropEvent`），与"打开图片"按钮等价。原方案正文未提，属补全。

### 修正 6：Alpha 后处理严格最小化（与规范一致，明确化）
仅做：clamp[0,1] → 清 NaN/Inf → 按 [0,255] 映射 → 高质量缩放回原尺寸（`INTER_LINEAR`，避免 `CUBIC` 过冲再 clip）。**不做**腐蚀/膨胀/羽化/锐化/去白黑边/填孔/删小区域。设置中的"清除极小杂点"默认关闭，**第一版不实现**（仅占位/隐藏）。

### 修正 7：Alpha 升采样质量
模型输出 1024² Alpha → 4K 时用 `INTER_LINEAR`；保留连续值，不二值化。如官方模型卡声明更高输出分辨率则按官方。

### 修正 8：框选 ROI 几何严格化
- 框选 → 按"框选宽高较大值的 5%（可调 0/5/10/15%）"四向扩展 padding → **clamp 到原图边界** → 从**原图**裁 ROI（非预览图）。
- 对 ROI letterbox 推理 → 裁 padding → Alpha 映射回 ROI 原尺寸 → 在原图坐标合成：**扩展框外 Alpha=0**，框内用模型 Alpha。
- 宽或高 < 50px → 提示"框选区域太小，请重新框选家具主体"，不推理。
- 不复用第一次整图蒙版。

### 修正 9：输出规则确认
PNG RGBA，8-bit，**Straight（非预乘）Alpha**。完全透明区 RGB 清零，半透明边缘保留原图 RGB。文件名 `原名_cutout.png`，默认存原图目录，不覆盖原图。可选 `原名_alpha.png`（默认关）。输出尺寸 == 原图尺寸。

### 修正 10：错误处理覆盖面
全部走 `QMessageBox`（非控制台）+ `logs/app.log`（`RotatingFileHandler`，含完整 traceback）。覆盖：模型加载失败 / 图片读取失败 / 内存不足(OOM) / 推理失败 / 模型下载失败 / 保存失败。失败后允许继续打开下一张。

### 修正 11：CPU 线程设置
`torch.set_num_threads(max(1, os.cpu_count()-2))`；`set_num_interop_threads(1)`。设置中可手动覆盖。不写死 12。

### 修正 12：测试分层
- **逻辑单测（无需模型，CI 可跑）**：`test_image_mapping.py`、`test_box_coordinates.py`、`test_export_size.py`、`test_alpha_composite.py` —— 覆盖坐标映射、padding 去除、输出尺寸、RGBA 合成。
- **模型冒烟测试（需模型，默认 skip）**：`test_engine_smoke.py`，`@pytest.mark.skipif(no model)`，用小合成图验证输出为 [0,1] 连续 Alpha 且尺寸回原图。
- **手动启动流程检查**：README 给出 checklist，按规范第 17 节逐项人工验证。

## 三、锁定的依赖版本（Python 3.11 64-bit，CPU-only）

> 实现时以"能装上且 BiRefNet trust_remote_code 可跑"为准；下列为初定锁定值，README 说明重装方法。

```
torch==2.3.1            # CPU, 安装自 https://download.pytorch.org/whl/cpu
transformers==4.45.2    # 兼容 BiRefNet trust_remote_code（若不兼容则回退 4.42.4，README 注明）
PySide6==6.7.3
Pillow==10.4.0
numpy==1.26.4
opencv-python-headless==4.10.0.84
huggingface-hub==0.25.1
pytest==8.3.2
```

`install.bat` 关键：检查 Python → 建 `.venv` → `pip install torch==2.3.1 --index-url https://download.pytorch.org/whl/cpu` → 装 requirements → 提示成功/失败。**不装 CUDA 版**。
