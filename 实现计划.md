## 阶段 1：BiRefNet 推理引擎（最高风险，最先做）

### 1.1 核实官方预处理（动手前必做）
- [P0] 读 HuggingFace `ZhengPeng7/BiRefNet_HR-matting` 模型卡
- [P0] 读 BiRefNet GitHub 仓库 `inference.py` / 官方 transform
- [P0] 记录：输入尺寸、mean/std、是否 letterbox、输出是否需 sigmoid、通道数与范围
- 产出：在 `birefnet_engine.py` 顶部注释固化官方来源与参数

### 1.2 birefnet_engine.py
- [P0] `BiRefNetEngine` 类：`load() → status, infer(rgb_pil) → alpha_np(0..1, H×W)`
- [P0] `device=cpu`、`model.eval()`、`torch.inference_mode()`
- [P0] float32，禁止 fp16/cuda
- [P0] letterbox 预处理：长边→模型尺寸 + pad 到正方形，记录 scale/padding
- [P0] 推理后：裁 padding → Alpha 高质量缩放回原图 H×W
- [P0] 输出连续 [0,1] Alpha：clamp + 清 NaN/Inf，**不二值化**
- [P0] 模型只加载一次，常驻内存
- [P0] `torch.set_num_threads(max(1, os.cpu_count()-2))`，可被设置覆盖
- [P1] 加载/推理进度回调钩子

**完成标准**：给定 PIL RGB，返回与原图同尺寸的 float [0,1] Alpha；二次调用不重建模型。

## 阶段 2：图像处理与输出

### 2.1 image_processing.py
- [P0] EXIF Orientation 应用（`ImageOps.exif_transpose`）
- [P0] 转 RGB（sRGB）
- [P0] 保留原始尺寸，建预览图（用于显示，不影响推理）
- [P0] RGBA 合成：原始 RGB + Alpha → RGBA（Straight，非预乘）
- [P0] 完全透明区 RGB 清零，半透明边缘保留原图 RGB
- [P0] letterbox 工具函数（供 engine 与 box 共用）：返回 padded + (scale, pad_l, pad_t)
- [P0] unletterbox 工具函数：去除 padding + 缩放回原尺寸

### 2.2 exporter.py
- [P0] 保存 PNG RGBA 8-bit Straight Alpha
- [P0] 文件名 `原名_cutout.png`，默认存原图目录，不覆盖原图
- [P0] 输出尺寸 == 原图尺寸
- [P1] 可选 `原名_alpha.png`（设置开关，默认关）
- [P0] 保存失败 → 抛可读异常（由 main_window 弹窗）

**完成标准**：`exporter.save(rgba, src_path, out_dir, also_alpha)→out_path`，输出尺寸=原图，PNG 含 alpha 通道。

### 4.4 settings.py + 设置对话框
