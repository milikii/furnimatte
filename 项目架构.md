# 项目架构 — Windows 极简家具自动抠图工具 (furniture_cutout)

> 本文是实现的唯一架构依据，配合 `可行性_与_优化方案.md` 与 `实现计划.md` 使用。
> 目标读者：负责单个模块的 worker（flash 级模型）—— 每个模块的**接口契约**必须在此明确，跨模块协作才不会错位。

---

## 1. 系统总览

纯本地、CPU-only、float32 的 Windows 桌面抠图工具。核心 = BiRefNet_HR-matting 推理 + 原尺寸 Alpha 映射 + 极简 PySide6 GUI。

```
┌─────────────────────────────────────────────────────────────┐
│  MainWindow (Qt 主线程)                                       │
│  ┌──────────┐  ┌────────────────┐  ┌──────────────────────┐  │
│  │ Toolbar  │  │ ImageView(左原图│  │ ImageView(右透明结果)│  │
│  │ 7 按钮   │  │ + BoxSelector) │  │ 棋盘/白/黑背景       │  │
│  └──────────┘  └────────────────┘  └──────────────────────┘  │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │ StatusBar: 状态/模型/尺寸/耗时 + 不确定进度条            │ │
│  └─────────────────────────────────────────────────────────┘ │
└──────────────────────┬──────────────────────────────────────┘
                       │ 信号/槽 (跨线程由 Qt 自动排队)
        ┌──────────────┴───────────────┐
        ▼                              ▼
┌───────────────────┐        ┌──────────────────────┐
│ InferenceWorker   │        │ BiRefNetEngine       │
│ (QThread 后台)    │───────▶│ (模型常驻, CPU/fp32) │
│ load/infer_full/  │        │ letterbox→推理→回映  │
│ infer_box         │        └──────────────────────┘
└───────────────────┘
        │ 调用
        ▼
┌───────────────────┐  ┌─────────────┐  ┌──────────┐
│ image_processing  │  │ exporter    │  │ settings │
│ EXIF/RGB/letterbox│  │ PNG RGBA    │  │ config   │
│ RGBA 合成         │  │ straight α  │  │          │
└───────────────────┘  └─────────────┘  └──────────┘
```

### 数据流（自动抠图）
```
用户打开图片
  → image_processing.load_image(path) → (original_rgb PIL, preview QImage, meta)
  → [点自动抠图] InferenceWorker.infer_full(original_rgb)
      → BiRefNetEngine.infer(rgb) → alpha_np (float [0,1], H×W)
  → image_processing.compose_rgba(original_rgb, alpha_np) → rgba PIL
  → ImageView 右侧显示 (棋盘格背景)
  → [点保存] exporter.save(rgba, src_path, out_dir, also_alpha) → out_path
```

### 数据流（框选重算）
```
用户点框选重算 → BoxSelector 在左图拖矩形 → 得到原图坐标 box
  → image_processing.expand_box(box, pad_ratio, img_size) → expanded_box (clamp 边界)
  → 若 box 宽或高 < 50px → 提示并终止
  → image_processing.crop_roi(original_rgb, expanded_box) → roi_pil
  → InferenceWorker.infer_box(roi_pil, expanded_box, img_size)
      → BiRefNetEngine.infer(roi_pil) → roi_alpha (ROI 原尺寸)
      → image_processing.map_roi_alpha_to_full(roi_alpha, expanded_box, img_size)
          → full_alpha (框外=0, 框内=roi_alpha)
  → image_processing.compose_rgba(original_rgb, full_alpha) → rgba
  → 显示 / 保存
```

## 2. 模块职责与接口契约

> 每个 worker 只实现自己模块；**禁止跨模块改文件**。接口签名即契约，先写函数签名+docstring 再实现。

### 2.1 `furniture_cutout/image_processing.py`
| 函数 | 签名 | 契约 |
|---|---|---|
| `load_image(path)` | `→ tuple[PIL.Image, dict]` | 应用 EXIF Orientation；转 RGB；保留原始尺寸；返回 (rgb_pil, meta={width,height,path})。meta 供状态栏显示。 |
| `make_preview(rgb_pil, max_side=1024)` | `→ PIL.Image` | 等比缩放预览图，仅用于显示，不参与推理。 |
| `letterbox(img_pil, target, pad_value)` | `→ tuple[PIL.Image, float, int, int]` | 长边→target 等比缩放，短边 pad 到正方形；返回 (padded, scale, pad_left, pad_top)。 |
| `unletterbox_mask(mask_np, orig_hw, scale, pad_lt)` | `→ np.ndarray float[0,1] (H×W)` | 裁掉 padding → 按原比例高质量缩放回 orig H×W。 |
| `expand_box(box, pad_ratio, img_size)` | `→ tuple[int,int,int,int]` | 按框宽高较大值×pad_ratio 四向扩展，clamp 到 [0,img_w]/[0,img_h]。 |
| `crop_roi(rgb_pil, box)` | `→ PIL.Image` | 从原图裁 ROI（非预览图）。 |
| `map_roi_alpha_to_full(roi_alpha, box, img_size)` | `→ np.ndarray (H×W)` | 框外=0，框内贴 roi_alpha（同尺寸直接放回）。 |
| `compose_rgba(rgb_pil, alpha_np)` | `→ PIL.Image RGBA` | Straight（非预乘）Alpha；透明区 RGB=0，半透明边缘保留原 RGB；输出尺寸=原图。 |
| `qimage_from_pil(pil_img)` | `→ QImage` | PIL → QImage 转换（供 ImageView）。 |
| `checkerboard(w,h,cell=16)` | `→ QImage` | 棋盘格背景。 |

**禁止**：对 Alpha 二值化、腐蚀/膨胀/羽化/锐化/填孔/删小区域。

### 2.2 `furniture_cutout/birefnet_engine.py`
| 函数 | 签名 | 契约 |
|---|---|---|
| `BiRefNetEngine.__init__(model_id, cache_dir, num_threads, progress_cb)` | | 不在构造里加载（避免阻塞）。 |
| `load()` | `→ None` (可抛) | `AutoModelForImageSegmentation.from_pretrained(model_id, trust_remote_code=True, cache_dir=...)`；`to(cpu)`；`eval()`；`torch.set_num_threads(num_threads)`；`torch.set_num_interop_threads(1)`。 |
| `infer(rgb_pil)` | `→ np.ndarray float[0,1] (H×W)` | 输入 PIL RGB 原图尺寸；内部 letterbox 到模型尺寸 → `torch.inference_mode()` float32 推理 → unletterbox 回原图尺寸 → clamp[0,1]+清 NaN/Inf。**输出 H×W == 输入 H×W**。 |
| `is_loaded` | property | 模型是否已加载。 |
| `MODEL_INPUT_SIZE` | 常量 | 默认 1024；**以官方模型卡为准**（实现前由编排者核实并填入）。 |

**官方预处理参数（mean/std/是否 sigmoid）由编排者核实后填入本节**，worker 不得凭空编造。
> 占位（T05 执行前由编排者确认替换）：
> - 输入尺寸：1024×1024（letterbox，非拉伸）
> - 归一化：ImageNet mean=[0.485,0.456,0.406] std=[0.229,0.224,0.225]（待官方核实）
> - 输出：`torch.sigmoid(logits)[0,0]` → [0,1]（待官方核实）

**禁止**：fp16、`torch.cuda`、每图重建模型、检测不到 CUDA 报错。

### 2.3 `furniture_cutout/inference_worker.py`
`InferenceWorker(QThread)`，信号：
- `status(str)` — 状态文本
- `progress(str)` — 处理中提示
- `finished(result)` — result={"rgba": PIL, "alpha": np.ndarray, "elapsed": float, "mode": "full"|"box"}
- `failed(kind:str, message:str, traceback:str)` — kind ∈ {model_load, image_read, oom, inference, download, save}

槽/方法（主线程调用，内部排队到线程）：
- `request_load()` — 首次加载模型
- `request_infer_full(rgb_pil)`
- `request_infer_box(rgb_pil, box)` — 内部调 image_processing.expand_box/crop_roi/map_roi_alpha_to_full
- 同一时刻只跑一个推理；已在跑则不发新任务（主线程侧用按钮禁用保证）

**禁止**：在主线程加载/推理。

### 2.4 `furniture_cutout/image_view.py`
`ImageView(QWidget)`：
- `set_original(pil)` / `set_result(rgba_pil)`
- `set_background(mode)` — mode ∈ {"checker","white","black"}
- 支持缩放（滚轮）+ 拖动（中键/左键拖动画布）；保留**视图坐标→原图坐标**换算（供 BoxSelector）。
- `view_to_image(pos)→(x,y)` 原图坐标

### 2.5 `furniture_cutout/box_selector.py`
`BoxSelector` 安装在左侧 ImageView 上：
- 鼠标按下→拖动→松开，画半透明矩形
- `box_drawn = pyqtSignal(int,int,int,int)` — 原图坐标 (x,y,w,h)
- 小框保护（<50px）在 main_window 判定（用消息框提示），selector 只负责产出坐标

### 2.6 `furniture_cutout/settings.py`
- `Settings` dataclass：model_id, model_cache_dir, box_pad_ratio, cpu_threads(int or "auto"), output_dir, save_alpha(bool), auto_cutout_on_open(bool), hf_mirror(bool)
- `load()/save()` 读写 `config.json`（项目根）
- `SettingsDialog(QDialog)`：上述字段表单

### 2.7 `furniture_cutout/exporter.py`
| 函数 | 签名 | 契约 |
|---|---|---|
| `save(rgba_pil, src_path, out_dir=None, also_alpha=False)` | `→ str(out_path)` | PNG RGBA 8-bit **Straight** α；文件名 `<stem>_cutout.png`；默认存原图目录；不覆盖原图（若重名则 `<stem>_cutout_1.png`）；输出尺寸=原图。 |
| `save_alpha(alpha_np, src_path, out_dir)` | `→ str` | 灰度 PNG `<stem>_alpha.png`。 |

失败抛 `ExporterError`（可读 message）。

### 2.8 `furniture_cutout/logging_config.py`
- `setup_logging(log_dir="logs")` → `RotatingFileHandler` app.log + 控制台镜像
- `log_exception(exc)` → 完整 traceback 入文件

### 2.9 `app.py`
入口：`setup_logging()` → `QApplication` → `MainWindow` → `sys.exit(app.exec())`；全局异常兜底（写日志 + QMessageBox）。

## 3. 线程模型

| 线程 | 职责 | 禁止 |
|---|---|---|
| Qt 主线程 | GUI 事件、状态栏、按钮启停、错误弹窗、显示结果 | 模型加载/推理、大图拷贝 |
| InferenceWorker (QThread) | 模型加载、推理、Alpha 映射、RGBA 合成 | 直接操作 widget（只发信号） |

跨线程通信**只用信号/槽**（Qt 自动 QueuedConnection）。worker 完成后把结果 PIL/numpy 发回主线程显示。

CPU 线程：`num_threads = settings.cpu_threads or max(1, os.cpu_count()-2)`；`set_num_interop_threads(1)`。

## 4. 坐标系（最易出错，单独成节）

三个坐标系：
1. **原图坐标** (0..img_w, 0..img_h) — 一切推理/保存/框选的权威坐标。
2. **预览/视图坐标** (0..view_w, 0..view_h) — ImageView 显示坐标，含缩放与平移。
3. **模型坐标** (0..MODEL_INPUT_SIZE) — letterbox 后，仅 engine 内部使用。

换算链：
- 框选：视图坐标 → `ImageView.view_to_image()` → 原图坐标 box。**BoxSelector 必须用此换算**，缩放/拖动画布后仍准确。
- 推理：原图 → letterbox → 模型 → unletterbox 回原图。letterbox 的 (scale, pad_left, pad_top) 必须在 engine 内部记录并用于 unletterbox，**不得丢失**。
- 框选 ROI：原图 box → expand_box → crop_roi → 推理 → map_roi_alpha_to_full 回原图。

## 5. 预处理与 Alpha 管线（核心质量逻辑）

```
原图 PIL RGB (H×W)
  → letterbox(1024) → padded (1024×1024) + (scale, pad_l, pad_t)
  → to_tensor / 归一化(官方参数) → (1,3,1024,1024) float32
  → torch.inference_mode(): model(input)
  → logits → [sigmoid] → alpha_model (1,1,1024,1024) float32 [0,1]
  → unletterbox_mask(alpha_model, (H,W), scale, pad_l, pad_t)
      → 裁掉 padding → cv2.resize(INTER_LINEAR) → (H,W) [0,1]
  → np.nan_to_num + np.clip(0,1)
  → 返回 alpha_np (H×W == 原图)
```

**质量红线**：
- 输出尺寸严格 == 原图（不得 2048 缩小导出）
- Alpha 连续 [0,1]，不二值化
- 原始 RGB 不被模型修改（合成时取原图 RGB）
- 不做腐蚀/羽化/填孔等后处理

## 6. 错误模型

| 错误类 | kind | 用户提示 | 处理 |
|---|---|---|---|
| 模型加载失败 | model_load | "模型加载失败，请检查模型目录或网络" | 不退出；可改设置选本地目录 |
| 模型下载失败 | download | "无法下载模型。请检查网络/代理或手动选择本地模型目录" | 不退出 |
| 图片读取失败 | image_read | "图片读取失败：<文件>" | 不退出；可打开下一张 |
| 内存不足 | oom | "内存不足，请尝试较小图片或关闭其他程序" | 不退出 |
| 推理失败 | inference | "推理失败，详情见日志" | 不退出 |
| 保存失败 | save | "保存失败：<路径>" | 不退出 |

全部：QMessageBox（非控制台）+ `logs/app.log` 完整 traceback。**禁止 `except: pass` 吞错误**。

## 9. 关键不变量（验收锚点）

1. 输出 PNG 尺寸 == 原图尺寸
2. 输出含 8-bit Straight Alpha
3. 原始家具 RGB 未被生成式重绘
4. 模型只加载一次（第二次推理无 reload）
5. 推理期间 GUI 可移动/刷新不卡死
6. 框选 ROI 坐标在缩放/拖动后准确；框外完全透明
7. 无 CUDA 环境可启动并推理
8. 失败有可读弹窗 + 日志 traceback，不崩溃退出
